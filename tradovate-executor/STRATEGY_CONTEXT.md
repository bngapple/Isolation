# Strategy Context — Hybrid v2 Parity (MNQ Only)

This document describes the exact Python code that produced
`reports/backtests/hybrid_v2_parity_mnq_only.json`.
Written for an AI agent working in this directory.

---

## The one file that matters

**`run_hybrid_v2_parity.py`** — self-contained parity runner.
It does NOT use `signal_engine.py`, `backtest_engine.py`, or `config.py`.
Everything — indicators, signal logic, backtest loop, MC — lives inside this
single file. The JSON was generated by running `main()` in this file.

## Data Pipeline

1. Load 1-minute MNQ parquet from `data/processed/MNQ/1m/`
2. `resample_15m_session(minute_df, end_hhmm=1600)` — filters RTH bars
   (`hhmm >= 930` and `hhmm < 1600`), then group_by_dynamic every 15m → OHLCV
3. All three strategies receive the same `df_15m` DataFrame
4. Each strategy produces a `np.ndarray` of signals (`+1` BUY, `-1` SELL, `0` flat)
5. All three signal arrays feed into `backtest()` independently
6. Trade lists are merged and sorted, then `calc_metrics()` + `run_mc()` produce the JSON

## Strategy 1 — RSI Extremes (`sig_rsi_extreme`, line 132)

**Indicator**: `calc_rsi(closes, period=5)` — Wilder smoothing on 15m closes.

```python
if rsi[i] < 35:   sigs[i] =  1   # BUY  (oversold)
elif rsi[i] > 65: sigs[i] = -1   # SELL (overbought)
```

**Trade rules**: SL=10pts, TP=100pts, max_hold=5 bars (75 min), 3 contracts
**Character**: Mean-reversion. High frequency (~6.6 trades/day). 22% win rate.
Largest contributor — 57% of total PnL ($318,943 of $561,768).

## Strategy 2 — IB Breakout (`sig_ib_breakout`, line 146)

**IB window**: bars where `hhmm < 1000` (9:30–10:00, first two 15m bars).
IB high/low built daily from those bars.

**Filters before signaling**:
1. `hhmm < 1000` or `hhmm >= 1530` → skip (only trade 10:00–15:30 ET)
2. Already traded today → skip (max 1 IB trade per day)
3. `ib_range < 2.0` → skip (too narrow)
4. Range filter (`ib_filter=True`): IB range must fall within P25–P75 of the
   trailing 50 days' IB ranges. If outside, no trade.

```python
if highs[i] > ib_h:  sigs[i] =  1   # BUY breakout
elif lows[i] < ib_l: sigs[i] = -1   # SELL breakdown
traded[day] = True                   # one trade per day, locks out immediately
```

**Trade rules**: SL=10pts, TP=120pts, max_hold=15 bars (225 min), 3 contracts
**Character**: Breakout, ~1 trade/day. Lowest win rate (21%) and weakest PnL
contributor (9%, $50,637). Most reliable month consistency (62/84 positive).

## Strategy 3 — Momentum Bars (`sig_momentum_bar`, line 199)

**Indicators**:
- `calc_atr(highs, lows, closes, 14)` — Wilder ATR on 15m bars
- `calc_ema(closes, 21)` — standard EMA(21)
- `avg_vol[i] = mean(volumes[i-20:i])` — rolling 20-bar volume average

**Three conditions must all be true**:
```python
bar_range = highs[i] - lows[i]
bar_range  > atr[i]  * 1.0    # condition 1: wide bar
volumes[i] > avg_vol[i] * 1.0 # condition 2: above-avg volume
```

**Direction**:
```python
bar_dir   = 1 if closes[i] > opens[i] else -1   # bar color
trend_dir = 1 if closes[i] > ema21[i] else -1   # above/below EMA

if bar_dir == trend_dir:
    sigs[i] = bar_dir   # only trade when bar agrees with trend
```
Both conditions must point the same way — no counter-trend trades.

**Trade rules**: SL=15pts, TP=100pts, max_hold=5 bars (75 min), 3 contracts
**Character**: Trend-following, ~2.7 trades/day. Best win rate (27.5%).
Second contributor — 34% of PnL ($192,188).

## Backtest Loop (`backtest()`, line 228)

Runs once per strategy. Key mechanics:

```
signal on bar i  →  pending = signal direction
bar i+1:
  - if hhmm >= flatten_time-15 (16:30): cancel pending, no entry
  - else: entry_px = opens[i+1] + direction * SLIP_PTS

while in position (bar j > entry_bar):
  - if hhmm[j] >= 1645: force exit at close  (EOD)
  - if long:  lows[j]  <= stop_px  → SL exit
              highs[j] >= target_px → TP exit
  - if short: highs[j] >= stop_px  → SL exit
              lows[j]  <= target_px → TP exit
  - if bars_held >= max_hold: exit at close  (max hold)

pnl = (exit_px - entry_px) * direction * POINT_VALUE * contracts - rt_cost
```

Exits are checked at the **bar level** (not tick), using high/low as proxies.
Slippage applied again on exit: `exit_px -= direction * SLIP_PTS`

## Monte Carlo (`run_mc()`, line 368)

```python
# Baseline:      pnl_mult=1.0  (raw backtest PnL)
# Conservative:  pnl_mult=0.70 (30% haircut on all trade PnLs)

for sim in 5000:
    shuffle daily buckets randomly
    walk forward day by day:
        cap any single day at DAILY_LIMIT (-$3000)
        if cumulative - peak <= MLL (-$4500): blown up, stop
        if cumulative >= EVAL_TARGET ($9000):  passed, record days taken
```

Results:
- Baseline:     83.1% pass, 16.9% blowup, median 28 days
- Conservative: 99.3% pass,  0.7% blowup, median 40 days

## Position Model — Critical Assumption

All three strategies run **fully independently with overlapping positions**.
Maximum exposure = 9 MNQ contracts simultaneously (3 per strategy).
There is NO conflict resolution, no shared risk tracking, no daily limit
enforcement between strategies in this parity runner.

This is explicitly flagged in the JSON: `"assumption": "independent overlapping strategy positions"`

**This is NOT how live execution works.** The live engine (`backtest_engine.py`
+ `signal_engine.py`) enforces single-position-per-account and uses
`ExecutionConfig.strategy_edge_scores` to resolve conflicts.
The parity runner exists purely to reproduce original research numbers.

```bash
cd tradovate-executor
python run_hybrid_v2_parity.py \
  --data data/processed/MNQ/1m/ \
  --output reports/backtests/hybrid_v2_parity_mnq_only.json
```

Optional flags:
- `--session-end 1600` (default) — last minute bar included
- `--flatten-time 1645` (default) — EOD flatten HHMM
